# Post와 PostComment는 소프트 삭제한다

게시판에서 글·댓글을 물리 삭제하지 않고 `deleted` 플래그로 소프트 삭제하기로 했다.

## 배경

대댓글이 1단계 허용된다. 답글이 달린 부모 댓글을 물리 삭제하면 대화 맥락(밑에 달린 대댓글)이 통째로 사라진다. 사용자는 "부모 댓글 본문은 '삭제된 댓글입니다'로 바꾸되 대댓글은 계속 보여야 한다"를 요구했고, 일관성을 위해 대댓글이 없는 댓글, 나아가 Post까지 동일하게 소프트 삭제하기로 정했다.

## 결정

- Post, PostComment 모두 물리 삭제하지 않고 `deleted=true`로 표시한다.
- 삭제된 PostComment: 본문 대신 "삭제된 댓글입니다"를 보여주고, 하위 대댓글은 계속 노출한다.
- 삭제된 Post: 목록·상세에서 숨기지 않고 "삭제된 글" 플레이스홀더로 제자리에 렌더한다.
- 즉 조회/목록은 필터링하지 않고 플레이스홀더로 노출하되, 삭제된 대상에는 수정·댓글 작성·추천 등 변경 행위를 차단한다.

## 고려한 대안

- **물리 삭제 + cascade**: 가장 단순하지만 답글 달린 부모 댓글 삭제 시 스레드가 사라져 요구사항 위반.
- **댓글만 소프트 삭제, Post는 물리 삭제**: 도메인 안에 삭제 정책이 두 개 공존 → 쿼리·검증 로직이 엔티티마다 갈려 혼란.

## 결과

- 장점: 삭제 후에도 스레드 구조·추천·조회수 이력 보존, 복구·감사 용이.
- 비용: 변경 행위(수정·댓글·추천)마다 대상의 `deleted` 여부를 검증해야 한다. 삭제된 대상이 목록·검색·페이징에 계속 섞여 보이므로 UX상 어색할 수 있고, 물리 정리(purge) 정책은 별도 과제로 남는다.
